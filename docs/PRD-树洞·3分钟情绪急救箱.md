# 产品PRD元提示词：树洞·3分钟情绪急救箱

## 1. 一句话产品定位

专为14-30岁青少年设计的"情绪急救工具"——当情绪上头时，选一个情绪标签，系统给出3个30秒能做完的具体动作，先止痛再谈长程。

## 2. 核心用户画像

### 主要用户群体
- **大学生/研究生**：考试焦虑、论文压力、人际关系
- **初入职场1-3年**：加班焦虑、自我怀疑、社交疲惫
- **异地漂泊年轻人**：远离家人、孤独感强、无人倾诉

### 代表用户故事
**小林（22岁大三学生）**

晚上11点，小林刚被导师退回第二稿论文，室友都在打游戏没人说话。她盯着屏幕发呆，胸口像压了块石头：想哭但哭不出来，想找人聊但不知道找谁——和爸妈说怕他们担心，和朋友说怕"传播负能量"，预约心理咨询要等下周。

**以前**：刷短视频熬到凌晨1点还是难受 / 写日记写到一半觉得矫情就关了 / 蒙着被子假装睡了

**现在**：打开树洞，点击"焦虑"，系统给了3个动作，她选了"给老朋友发一条微信"。朋友秒回"我也是"，她笑了。

## 3. 核心用户旅程

```
步骤1：打开页面（触达路径：高校公众号/B站UP主/班级群）
    ↓
步骤2：选择情绪类型（焦虑/愤怒/孤独/低落/迷茫）
    ↓
步骤3：调节情绪程度（1-5级滑动条）
    ↓
步骤4：获取个性化动作推荐（根据情绪+程度智能匹配）
    ↓
步骤5：执行动作（深呼吸/发消息/听音乐等）
    ↓
步骤6：匿名留言（可选，留下痕迹但不存储）
    ↓
步骤7：浏览树洞消息墙（看到其他人也在经历类似的情绪）
```

## 4. 功能清单

### Must Have（核心功能）
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 情绪选择 | 5种情绪卡牌（焦虑/愤怒/孤独/低落/迷茫），带颜色区分和icon | P0 |
| 情绪程度调节 | 1-5级滑动条，影响动作推荐内容 | P0 |
| 动作推荐 | 根据情绪+程度智能推荐3个动作，每个动作含描述和时间 | P0 |
| 深呼吸引导 | 60秒4-7-8呼吸法动画（圆圈缩放+计时器+进度条） | P0 |
| 匿名留言 | 底部"留一句话"输入框，提交后显示"🌱已收到" | P1 |
| 树洞消息墙 | 展示其他用户匿名留言（模拟数据），建立"不孤独"感受 | P1 |
| 情绪日记 | 7天情绪折线图+统计数据（使用次数/连续天数/今日情绪） | P2 |
| 音乐推荐 | 4种心情（平静/治愈/振奋/释放）各3首歌单 | P2 |

### Nice to Have（增强功能）
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 消息推送提醒 | 每天固定时间推送"今天感觉怎么样？" | P3 |
| 成就系统 | 连续使用7天解锁"坚持者"徽章 | P3 |
| 分享功能 | 将某个动作分享给朋友 | P3 |

### Won't Do（绝对不做）
| 功能 | 原因 |
|------|------|
| 用户注册/登录系统 | 纯工具原型无需登录，匿名是核心体验 |
| 后端API/数据库 | 所有数据本地存储，不上传服务器 |
| 真实音乐播放 | 接入真实API会增加复杂度，用alert模拟即可 |
| 社交功能 | 纯匿名工具，不做互动/评论/点赞 |
| 商业变现 | 创意赛不要求盈利 |

## 5. 界面与交互设计要点

### 视觉风格
- **整体调性**：温暖治愈、安全感、不评判
- **配色方案**：莫兰迪色系低饱和度，避免刺激情绪
  - 主色：#8B9A6B（橄榄绿）
  - 焦虑：#A8C5D9（雾霾蓝）
  - 愤怒：#D4A5A5（玫瑰粉）
  - 孤独：#B5D8EB（天空蓝）
  - 低落：#C9B8A8（燕麦色）
  - 迷茫：#B8A9C9（薰衣草紫）
  - 背景：#F5F3F0（暖白色）
- **字体**：系统默认字体，14-16px为主

### 布局结构
```
┌─────────────────────────┐
│     🌳 树洞·情绪急救箱    │  Header（品牌标识）
├─────────────────────────┤
│  [急救] [树洞] [日记] [音乐] │  Nav Tabs（4个Tab切换）
├─────────────────────────┤
│                         │
│    💙 焦虑              │  Emotion Card
│    💔 愤怒              │  Emotion Card
│    🤍 孤独              │  Emotion Card
│    💛 低落              │  Emotion Card
│    💜 迷茫              │  Emotion Card
│                         │
├─────────────────────────┤
│  [─────●─────]          │  Intensity Slider
│   一点点 ── 非常强烈     │
│  [给我推荐动作]          │  CTA Button
└─────────────────────────┘
```

### 记忆点交互
1. **情绪卡牌悬停**：轻微上浮+背景色渐变
2. **深呼吸引导**：圆圈呼吸动画（吸气放大/呼气缩小）
3. **匿名留言反馈**：提交后只显示"🌱已收到"，极简不打扰
4. **消息墙情感共鸣**：看到"其他人也在经历类似的情绪"

## 6. 数据与内容策略

### 数据类型
| 数据类型 | 内容来源 | 存储方式 |
|----------|----------|----------|
| 情绪类型 | 内置5种情绪 | JS常量 |
| 动作推荐 | 根据情绪+程度算法生成 | JS常量+随机 |
| 树洞留言 | 模拟6条真实感留言 | JS数组 |
| 情绪日记 | 模拟7天数据 | JS对象 |
| 音乐歌单 | 4种心情各3首 | JS数组 |

### 模拟数据示例
```javascript
const messageWallData = [
  { emotion: 'lonely', text: '今天一个人吃完了一整盒外卖，突然不知道自己在干什么。', time: '3分钟前' },
  { emotion: 'anxiety', text: '论文又被导师退回来了，已经是第三次了...', time: '15分钟前' },
  { emotion: 'low', text: '每天醒来都觉得好累，不知道这样的日子还要持续多久。', time: '32分钟前' }
];
```

### 样例数据要求
- 留言要真实、有代入感，像真实用户写的
- 不要过于煽情或"正能量"，保持"被理解"的调性
- 时间分布要有层次感（刚刚/3分钟前/1小时前/昨天）

## 10. 关键技术实现说明

### HTML结构
- 单文件HTML，内嵌CSS和JavaScript
- 无外部依赖，可直接浏览器打开
- 移动端优先设计，max-width: 480px

### CSS特性
- CSS Variables管理颜色系统
- Flexbox布局
- CSS Animations（gentlePulse/breatheCircle/fadeIn）
- 响应式设计（@media max-width: 360px）

### JavaScript功能
- 状态管理（currentEmotion/currentIntensity）
- Tab切换（switchTab）
- 动态渲染（renderMessageWall/renderMusicList）
- 呼吸动画定时器（breathingInterval）
- 情绪程度滑动条（updateIntensity）

### 本地存储
- 暂不使用localStorage（保持完全匿名）
- 如需持久化可添加：localStorage.setItem('emotion_diary', JSON.stringify(data))
